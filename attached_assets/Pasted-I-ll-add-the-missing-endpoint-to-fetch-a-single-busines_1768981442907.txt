2. Approve Business Application
POST /api/admin/businesses/:id/approve
Authorization: Admin required
Response:

{
  "success": true,

Notifications Sent:

Email to owner with Stripe onboarding CTA (if applicable)

POST /api/admin/businesses/:id/reject
Authorization: Admin required
Body: { "reason": "Reason for rejection" }
Response:

{
  "success": true,
  "business": { /* updated business object */ },
  "message": "Business rejected successfully"
}
Notifications Sent:

In-app notification to business owner with rejection reason
Email to owner explaining rejection and reapply options
