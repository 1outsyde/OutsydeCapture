2. Storefront Visibility Rules
Business approvalStatus Field (shared/schema.ts line 196)
// businesses table
approvalStatus: text("approval_status").default("pending").notNull(),
approvalNotes: text("approval_notes"),
approvedAt: timestamp("approved_at"),
approvedBy: varchar("approved_by", { length: 36 }),
Approval Status Values:
Status	Meaning
pending	New vendor application, awaiting admin review
approved	Admin approved, storefront visible to customers
rejected	Admin rejected, storefront hidden
Admin Endpoints:
GET  /api/admin/applications?status=pending   - List pending applications
POST /api/admin/applications/:id/approve      - Approve (notes optional)
POST /api/admin/applications/:id/reject       - Reject (notes required)
Note: There is no separate storefrontStatus field. Visibility is controlled by approvalStatus on the businesses table.

3. Subscription Tier Publishing Gates
Publishing Requirements (Products & Services)
To publish (Go Live) a product or service, vendors must have:

Stripe Connect Onboarding Complete

business.stripeAccountId must exist
business.stripeOnboardingComplete must be true
Active Subscription

Checked via storage.isBusinessSubscriptionActive(businessId)
Returns { active: boolean, status?: string, reason?: string }
Subscription Status Logic (server/storage.ts lines 3017-3027)
// Active statuses
'active' → active: true
'trialing' → active: true
'past_due' → active: true (within 3-day grace period)
// Inactive statuses  
'canceled', 'unpaid', 'incomplete', 'incomplete_expired' → active: false
No subscription → active: false
API Response Errors:
Missing Stripe Onboarding:

{
  "error": "Payments not enabled",
  "message": "You must complete Stripe payment setup before publishing products/services.",
  "requiresOnboarding": true
}
Missing Subscription:

{
  "error": "Subscription required", 
  "message": "You must have an active subscription to publish products/services.",
  "requiresSubscription": true
}
Editing vs Publishing:
Action	Requires Subscription?	Requires Stripe Onboarding?
Create draft product/service	No	No
Edit draft product/service	No	No
Publish (Go Live)	Yes	Yes
