Now to answer your earlier question about closed days format for the photographer availability endpoint:

The PUT /api/photographers/me/availability endpoint expects the hoursOfOperation field in this format:

{
  "hoursOfOperation": {
    "monday": { "open": "09:00", "close": "17:00" },
    "tuesday": { "open": "09:00", "close": "17:00", "closed": true },
    "wednesday": { "open": "09:00", "close": "17:00" }
  }
}
For closed days, you should use:

{ "open": "09:00", "close": "17:00", "closed": true } - The closed: true flag marks the day as unavailable
Days that are simply omitted from the object will fall back to whatever default behavior exists (typically treated as unavailable). The format { monday: null } would fail validation since the schema requires open and close strings.
